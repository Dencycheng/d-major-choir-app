# v0.4 架构说明

v0.4 不改变 v0.3 的主体架构，重点补齐管理后台和小程序端接入。

## 后端

- FastAPI 按业务路由拆分。
- SQLAlchemy 管理核心业务模型。
- JWT 认证。
- `require_member` 现在仅允许 active 成员访问合唱团内部数据。
- pending 成员仅能在“我的合唱团”中看到申请关系，不能进入活动、任务、谱库等内部模块。

新增接口：

```http
GET /api/choirs/{choir_id}/practice-records
```

用途：管理后台汇总查看全团练习打卡，可通过 `task_id` 查询参数筛选。

## 管理后台

管理后台目前仍是轻量联调版本，不追求最终 UI，但已覆盖真实业务动作：

- 成员审批
- 请假审批
- 活动创建和签到码生成
- 作品创建
- 文件上传并绑定为作品资料
- 练习任务发布
- 打卡记录查看与点评

## 小程序

小程序端新增统一请求封装：

```text
miniapp/utils/api.js
```

当前页面已可读取真实后端数据，但录音上传仍为 Demo URL。v0.5 建议接入微信录音能力和文件上传能力。

## 已知限制

1. 后台文件上传目前直接返回 `/uploads/<filename>`，文件仍是静态公开访问。
2. 管理后台尚未做完整表单校验。
3. 小程序端尚未做真实录音上传。
4. 管理后台尚未接入路由系统，当前为单页 Tabs。
5. 尚未增加 CI/CD。
