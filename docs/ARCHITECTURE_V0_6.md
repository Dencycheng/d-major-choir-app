# v0.6 架构说明

v0.6 的目标是把 MVP 从“可部署”继续推进到“可试点使用”。本版本没有改变核心数据库结构，主要补齐管理后台和小程序端在真实排练场景中的体验缺口。

## 重点变化

### 1. 活动与考勤体验

- 新增活动反馈统计接口：`GET /api/events/{event_id}/response-statistics`
- 新增签到二维码图片接口：`GET /api/events/{event_id}/checkin-qr.png`
- 管理后台可直接展示签到二维码，不再只弹出字符串签到码。
- 管理后台可查看活动参加反馈、签到、请假等简要统计。

### 2. 声部长看板

- 新增接口：`GET /api/choirs/{choir_id}/section-dashboard`
- 管理后台新增“声部长看板”Tab。
- 指标包括：声部成员数、活跃成员数、相关任务数、打卡记录数、待点评数和出勤率。

### 3. 谱库与录音预览

- 管理后台对谱库资料和练习录音采用鉴权预览流程。
- 对 `/api/files/{asset_id}/download` 类型文件，先获取短期签名链接，再打开。
- 小程序谱库页面支持进入作品详情，查看资料列表，并根据文件类型打开 PDF 或播放音频。

### 4. 成员导入试点模板

- 新增成员导入模板接口：`GET /api/choirs/{choir_id}/members/import-template.csv`
- 管理后台可下载模板并上传成员 CSV。

### 5. 管理后台可用性增强

- 表单增加基础校验。
- 错误提示与成功提示区分颜色。
- 资源打开、录音打开、活动统计等动作从“复制链接”升级为可点击操作。

## 新增依赖

后端新增：

```text
qrcode[pil]
```

用于生成排练签到二维码 PNG。

## 未改变的部分

- 数据库模型未增加新表。
- Alembic 迁移无需新增版本。
- AI反馈仍为预留能力，不在 v0.6 中实现高精度算法。

## 下一步建议

v0.7 可以进入“权限与质量强化版”，重点包括：

1. 更细的权限单元测试。
2. 文件访问审计日志。
3. 成员端个人练习报告。
4. 小程序扫码签到真实路径。
5. 后台分页、搜索、筛选。
6. 生产环境对象存储适配。
