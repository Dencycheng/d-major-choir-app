# 测试账号（V2.1）

V2.1 起取消"无密码演示账号"。所有登录均为真实账号：

## 后台管理员

通过 `npm run create-admin` 创建（读取 `.env` 的 `ADMIN_EMAIL` / `ADMIN_MOBILE` / `ADMIN_PASSWORD`）：

| 项目 | 值 |
| --- | --- |
| 入口 | `https://admin.dmajorchoir.com` |
| 账号 | `admin@dmajorchoir.com`（按 .env 实际配置） |
| 初始密码 | `.env` 中 `ADMIN_PASSWORD`（首次登录强制改密） |
| 权限 | 超级管理员（全部权限 + 角色配置 + 审计日志） |

> 重置密码：`node scripts/create-admin.js --reset-password`（重新读取 .env 密码并恢复"首登改密"状态）。

## 小程序成员

不再预置账号。流程：

1. 后台「邀请与入团」生成邀请码（可限定声部/次数/有效期）。
2. 测试者用微信打开小程序 → 自动 `wx.login` → 进入「申请入团」页填写邀请码 + 资料。
3. 后台审核通过 → 成员自动建档并绑定微信，再次打开小程序即进入正式成员端。

## 种子演示数据

`npm run seed` 导入的成员（丁总监、许团务、林安等）仅为业务演示档案，**没有登录账号**，用于展示看板/谱库/任务数据。需要让某个演示档案可登录时，可在审核入团时绑定，或后续在成员管理中维护。

## 锁定策略

同一账号 10 分钟内密码错误 5 次将锁定 10 分钟；全部登录尝试记录在后台「登录日志」。
