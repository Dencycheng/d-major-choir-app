# V2.1 验收清单

执行环境：预发或生产（https://api.dmajorchoir.com）。自动化部分可直接跑 `scripts/smoke-test.js`（34 步全绿即覆盖大部分项）。

## A. 登录与账号安全

- [ ] 未登录访问任何业务接口（如 `/api/bootstrap`）返回 401
- [ ] 管理员邮箱/手机号 + 密码可登录后台
- [ ] 错误密码 5 次（10 分钟内）后账号锁定 10 分钟，提示明确
- [ ] 首次登录强制改密：未改密前业务接口返回 428，改密后旧 token 失效、新 token 可用
- [ ] 登出后原 token 立即不可用
- [ ] 后台「登录日志」可见成功/失败记录（含 IP、UA）
- [ ] 数据库 `users.password_hash` 为 scrypt 哈希，无明文；`.env` 不在 Git 中
- [ ] 小程序打开自动登录（wx.login），代码中无任何 Secret、无 X-Member-Id

## B. 邀请码与入团

- [ ] 后台可创建邀请码（声部/次数/有效期），可停用
- [ ] 小程序新用户进入「申请入团」页；无效/过期/停用邀请码提交报错
- [ ] 有效邀请码提交后显示"等待审核"；后台待审列表可见
- [ ] 审核通过后成员自动建档、绑定微信，小程序进入正式首页；驳回则显示留言并可重新申请
- [ ] 同一微信号已有待审申请时不可重复提交

## C. 权限与数据隔离

- [ ] 普通成员：看不到他人录音、他人请假理由、他人手机号明文（脱敏）
- [ ] 普通成员调用任务发布/邀请管理/看板/操作日志接口均 403
- [ ] 声部长仅能点评、审批、发任务到本声部
- [ ] 谱库资料按声部可见范围过滤；文件流裸地址（无 token）返回 401
- [ ] 后台「操作日志」记录关键写操作（上传、审核、点评、改密等）

## D. 业务闭环（保持 V1.2 能力）

- [ ] 活动：参加（"确认参加，期待一起唱歌"）/ 请假（需理由）/ 审批 / 签到（"签到成功，快快开嗓一起唱吧"）
- [ ] 谱库：上传 PDF/音频/视频，成员端可看可播，视频支持 0.75x/1x/1.25x/1.5x
- [ ] 练习：发布任务 → 成员录音打卡（"练习已收到，辛苦啦"）→ 自评 → 点评 → 成员可见反馈
- [ ] 看板：出勤率、打卡率、作品准备度正常聚合

## E. 数据与运维

- [ ] `npm run backup` 生成 SQLite + uploads 快照；`npm run restore` 可恢复
- [ ] 生产数据位于 `/home/ubuntu/d_major_runtime/`，`git pull` 不影响数据
- [ ] `/api/health` 返回 `version: 2.1.0`
- [ ] HTTPS 证书有效，HTTP 强制跳转 HTTPS
