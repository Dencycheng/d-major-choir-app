# MVP开发解析说明

## 当前开发策略

第一阶段只做三个闭环：

1. 活动考勤闭环
2. 练习打卡闭环
3. 谱库资料闭环

AI音准、节奏、气息分析先预留表 `ai_reports` 和字段 `practice_records.ai_status`，不阻塞第一版上线。

## 技术栈

- 后端：FastAPI + SQLAlchemy
- 数据库：SQLite本地开发，PostgreSQL生产部署
- 管理后台：React + Vite + TypeScript
- 小程序：微信原生小程序骨架

## 第一批UI页面

小程序：登录、首页、活动详情、签到、请假、任务详情、录音打卡、谱库详情、我的。

后台：数据看板、成员管理、活动管理、考勤统计、练习任务、打卡记录与点评、谱库管理。

## 权限重点

- 普通成员只能看自己的录音和点评
- 声部长只能看本声部数据
- 指挥/管理员可看全团
- 不同合唱团数据隔离
- 退出成员不能继续看内部资料

## 下一步开发优先级

1. 把当前单文件后端拆分为 routers / models / schemas / services。
2. 接入 Alembic 数据库迁移。
3. 完成 Web 后台 CRUD 页面。
4. 完成小程序登录、首页、任务和打卡页面。
5. 接入对象存储与签名 URL。
6. 替换 Demo Auth 为 JWT + 微信登录。
